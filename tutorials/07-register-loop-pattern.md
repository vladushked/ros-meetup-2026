# Tutorial 07: Паттерн `register_loop` для нескольких async-воркеров

## Проблема
Когда в одной ноде 4-6 фоновых задач, ручное управление ими становится шумным и ломким.

## Цель
Автоматически регистрировать loop-методы и запускать их единым раннером.

## Файл примера
- `ros-meetup-2026/examples/register_loop/task_orchestrator_node.py`

## Что показывает пример
1. Декоратор `register_loop` маркирует loop-методы.
2. Класс собирает отмеченные методы в `async_loops`.
3. `run()` запускает все loop параллельно.

Примеры loop в файле:
- `grasping_loop`
- `fsm_loop`
- `watchdog_loop`
- `ros_loop`

## Почему это решение принято
Так проще масштабировать ноду: добавление нового воркера сводится к одному `@register_loop` без переписывания orchestration-кода.
